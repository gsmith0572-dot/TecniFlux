Build a complete production-ready web application named TecniFlux. The purpose of this application is to allow automotive technicians to search, filter, decode VIN numbers, scan VIN labels, and view automotive wiring diagrams stored as PDF files in Google Drive. The structured index of all documents is stored in a Google Sheets file that is already created.

Follow all requirements below in full detail.

===================================================================

Purpose of the App
===================================================================

TecniFlux must allow technicians to:

Search automotive wiring diagrams by filters such as:

make

model

year

system

keyword

Search by VIN using:

manual input (17 characters)

scanning VIN barcodes (PDF417, Code 39)

capturing a photo of the VIN label and extracting text with OCR

Decode VIN information using the NHTSA VIN Decode API:
https://vpic.nhtsa.dot.gov/api/vehicles/decodevin/{VIN}?format=json

Open and view PDF wiring diagrams hosted in Google Drive using direct file URLs stored in Google Sheets.

===================================================================
2. Data Sources and Integration

A. Primary Database: Google Sheets
This Google Sheets file contains all indexed documents. Columns include:

file_name

file_id

file_url

direct_url

mime_type

parent_folder

full_path

last_updated

make

model

year

system

The application must:

Connect to Google Sheets via the Google Sheets API.

Load the entire dataset into the application.

Cache records for performance.

Filter and search records efficiently.

B. File Repository: Google Drive
All wiring diagrams are stored as PDF files in Google Drive.
The application must:

Use the "direct_url" field to open PDF files.

Display PDFs in an embedded PDF viewer component.

Support mobile and desktop PDF viewing.

3. User Authentication System

Implement a secure user auth system with:

Email and password login

JWT or session-based auth

Two user roles:

admin

technician

Route protection and secure API endpoints

Admins must be able to edit metadata but not upload PDFs.
Technicians can only search and view diagrams.
